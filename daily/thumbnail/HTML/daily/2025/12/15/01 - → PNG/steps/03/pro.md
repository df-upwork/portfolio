Для Cygwin необходимо использовать утилиту `cygpath` для преобразования путей POSIX (вида `/cygdrive/c/...`) в пути Windows.

Для протокола `file://` лучше всего подходит флаг `-m` (mixed), который создает путь Windows (с буквой диска), но использует прямые слеши (`/`), что корректно воспринимается браузером и не требует экранирования обратных слешей.

### Исправленный вариант

```bash
(
	o=(
		--headless=new
		--screenshot
		--window-size=1920,1080
	)
	chrome "${o[@]}" "file://$(cygpath -m "$(pwd)")/файл.html"
)
```

### Разбор флагов cygpath

  * **`cygpath -m`**: Преобразует в смешанный формат (Mixed): `C:/Users/Project`. Это лучший вариант для формирования URL.
  * **`cygpath -w`**: Преобразует в стандартный формат Windows: `C:\Users\Project`. В строке URL это может выглядеть как `file://C:\Users\Project/файл.html`, что иногда вызывает проблемы парсинга или требует экранирования.
  * **`cygpath -ma`**: (Рекомендуемый альтернативный вариант) Преобразует относительный путь сразу в абсолютный смешанный. Это позволяет передать имя файла целиком, а не склеивать строку.

### Альтернативный (более надежный) подход

Вместо склейки пути через `$(pwd)/файл`, безопаснее сразу запросить полный путь к конкретному файлу:

```bash
chrome "${o[@]}" "file://$(cygpath -ma "файл.html")"
```
