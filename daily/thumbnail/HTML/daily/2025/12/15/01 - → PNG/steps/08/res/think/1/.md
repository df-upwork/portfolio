1)  **Причина `P†`**
    Наблюдаемые проблемы вызваны двумя факторами, связанными с работой Chrome в режиме headless:

<!-- end list -->

  * **Смещение текста:** В системе запуска отсутствует локальный шрифт `Carlito`, указанный в `src: local(...)`. Браузер заменяет его стандартным шрифтом (например, Times New Roman), вертикальные метрики которого отличаются. Это приводит к визуальному смещению текста вниз относительно центра.
  * **Обрезанный сайдбар:** Фиксированная высота `height: 800px` в `body` при отсутствии явной высоты у `html` может приводить к некорректному расчету области фона в headless-режиме. Контейнер не растягивается на весь вьюпорт, из-за чего цветной сайдбар обрывается. Использование единиц `vh` (viewport height) устраняет эту проблему.

<!-- end list -->

2)  **Правки к `H⸙`**

# 1

В тег `<head>`, перед открывающим тегом `<style>`, добавь подключение веб-шрифта:

```html
	<link href='https://fonts.googleapis.com/css2?family=Carlito:wght@400;700&display=swap' rel='stylesheet'>
```

# 2

В блоке `<style>` удали оба правила `@font-face` (строки с 7 по 18 включительно), так как они замещаются внешним шрифтом.

# 3

В блоке `<style>` замени селектор `body { ... }` на следующий код (изменены `height`, `width` и добавлен фоллбек шрифта):

```css
		body {
			background-color: #ffffcc;
			display: grid;
			font-family: 'Carlito', sans-serif;
			grid-template-columns: 38.28px 1fr;
			grid-template-rows: 1fr;
			height: 100vh;
			margin: 0;
			overflow: hidden;
			padding: 0;
			width: 100vw;
		}
```
