1)  **Причина `P†`**
    Проблема вызвана тем, что ширина текста «alkaline hydrolysis» при заданном размере шрифта (\~106px) превышает ширину окна (1000px). В CSS Grid колонка с размером `1fr` по умолчанию ведет себя как `minmax(auto, 1fr)`, что заставляет сетку растягиваться под ширину содержимого.
    Это переполнение по ширине провоцирует появление (или резервирование места системой рендеринга) горизонтальной полосы прокрутки. Высота этой полосы вычитается из высоты области просмотра (`viewport`).
    В результате значение `100vh` оказывается меньше 800px (например, 783px), и блок `body` заканчивается раньше, чем нижний край скриншота, из-за чего фон боковой панели (`#sidebar`) выглядит обрезанным.

2)  **Правки к `H⸙` для устранения `P†`**

# 1

Вставь в начало блока `<style>` (перед селектором `body`) правило для элемента `html`, чтобы создать контекст для вычисления высоты в процентах.

```css
		html {

```

# 2

В селекторе `body` измени свойство `grid-template-columns`. Использование `minmax(0, 1fr)` ограничит ширину колонки доступным пространством и предотвратит растягивание сетки текстом.

```css
			grid-template-columns: 38.28px minmax(0, 1fr);
```
